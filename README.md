# 🔐 CertAuth System

인증서(PKI) 기반 + 안면인식(face-api.js) 로그인 시스템입니다.  
회원가입 시 디지털 인증서를 자동 발급하고, 이후 비밀번호 · 인증서 · 얼굴인식 세 가지 방법으로 로그인할 수 있습니다.

---

## 📋 전체 기능 목록

### 👤 회원가입 (4단계)

회원가입은 단계별로 진행되며, 각 단계를 완료해야 다음으로 넘어갈 수 있습니다.

#### STEP 1 — 개인정보 입력
- **이름**, **나이**, **이메일** 입력 (필수)
- 소속 부서 · 조직명 입력 (선택)
- 이메일 중복 확인 버튼으로 사용 가능 여부 즉시 검증

#### STEP 2 — 아이디 · 비밀번호 설정
- **아이디** 입력 — 영문, 숫자, 밑줄(_) 4~20자
- 아이디 중복 확인 버튼으로 즉시 검증
- **비밀번호** 입력 — 6자 이상 (강도 표시: 약함 / 보통 / 강함 / 매우 강함)
- 비밀번호 확인 입력
- 비밀번호 표시/숨기기 토글 버튼 지원

#### STEP 3 — 얼굴 등록 (선택)
- 웹캠으로 얼굴을 촬영하여 등록
- 등록된 얼굴로 이후 **카메라 한 번**으로 로그인 가능
- **건너뛰기** 버튼으로 나중에 등록 가능
- 등록 후 다시 촬영하여 교체 가능

#### STEP 4 — 인증서 발급
- RSA 2048-bit 키 쌍 자동 생성
- X.509 인증서 발급 (유효기간 1년)
- 발급된 인증서 정보 표시 (소유자, 시리얼 번호, 유효기간, SHA-256 핑거프린트)
- **PEM 파일 다운로드** 버튼으로 즉시 저장 가능
- ⚠️ PEM 파일을 반드시 안전한 곳에 보관하세요

#### STEP 5 — 완료
- 자동으로 로그인 처리 후 대시보드로 이동
- PEM 파일 재다운로드 가능

---

### 🔓 로그인 방법 3가지

#### 1. 비밀번호 로그인 탭
- 아이디(또는 이메일) + 비밀번호 입력
- 비밀번호 표시/숨기기 토글 지원

#### 2. 얼굴인식 로그인 탭
- **카메라 시작** 버튼 클릭
- 웹캠이 켜지면 얼굴을 화면 중앙에 맞추기
- 실시간으로 얼굴을 감지하여 자동 인증 (클릭 불필요)
- 초록 박스와 랜드마크로 감지 여부 시각적 확인 가능
- 인증 성공 시 자동으로 대시보드 이동

> 얼굴인식 로그인은 회원가입 시 또는 **얼굴 관리** 메뉴에서 얼굴을 미리 등록해야 사용 가능합니다.

#### 3. 인증서 로그인 탭
- 발급받은 PEM 파일을 드래그하거나 클릭하여 업로드
- 또는 PEM 텍스트를 직접 붙여넣기
- `.pem` · `.crt` · `.cer` 파일 형식 지원

### 🛡 관리자 페이지

`admin` 계정으로 로그인하면 추가 메뉴가 활성화됩니다.

#### 사용자 관리
- 전체 가입 사용자 목록 조회
  - 이름, 아이디, 나이, 이메일, 부서, 역할, 활성 상태, 얼굴 등록 여부 표시
- 이름 / 아이디 / 나이 / 이메일 / 부서로 실시간 검색
- **사용자 추가** — 관리자가 직접 계정 생성 (인증서 자동 발급)
- **인증서 발급** — 특정 사용자에게 새 인증서 재발급
- **활성화 / 비활성화** — 계정 접근 권한 즉시 전환
- **삭제** — 사용자 영구 삭제

#### 인증서 관리
- 발급된 모든 인증서 목록 조회
- 시리얼 번호, 상태, 발급일, 만료일, 핑거프린트 확인
- 사용자명 / 이메일 / 시리얼 번호로 실시간 검색
- **상세 보기** — PEM 내용, 핑거프린트 전체 확인
- **PEM 다운로드** — 해당 사용자의 인증서 파일 다운로드
- **폐기** — 사유 기록과 함께 인증서 즉시 폐기

#### 관리자 대시보드 통계
- 전체 / 활성 / 비활성 사용자 수
- 발급 / 유효 / 폐기 인증서 수
- 30일 내 만료 예정 인증서 수
- 얼굴 등록 사용자 수

## 🔐 인증 방식 상세 설명

### 인증서 (PKI) 작동 원리

```
회원가입
  └─→ 서버에서 RSA 2048-bit 키 쌍 생성
        └─→ X.509 형식 인증서 발급 (유효기간 1년)
              └─→ PEM 파일로 사용자에게 제공

인증서 로그인
  └─→ 사용자가 PEM 파일 업로드
        └─→ 서버가 인증서 파싱 → SHA-256 핑거프린트 추출
              └─→ DB 등록 핑거프린트와 대조
                    └─→ 일치 + 유효 + 미폐기 → 로그인 성공
```

### 얼굴인식 작동 원리

```
얼굴 등록
  └─→ 웹캠 영상에서 face-api.js가 얼굴 감지
        └─→ 128차원 descriptor(특징 벡터) 추출
              └─→ 서버 DB에 저장

얼굴인식 로그인
  └─→ 웹캠 실시간 영상에서 descriptor 추출
        └─→ DB에 등록된 모든 사용자 descriptor와 유클리드 거리 계산
              └─→ 가장 가까운 사용자 찾기
                    └─→ 거리 < 0.45 (임계값) → 로그인 성공
```

## ❓ 자주 묻는 질문

**Q. 얼굴인식이 작동하지 않아요.**  
A. 브라우저에서 카메라 권한을 허용했는지 확인하세요. Chrome 기준으로 주소창 왼쪽 자물쇠 아이콘 → 카메라 → 허용으로 설정하면 됩니다. `http://localhost:3000`에서만 정상 작동하며, `file://` 경로에서는 카메라가 차단됩니다.

**Q. PEM 파일을 잃어버렸어요.**  
A. 관리자에게 인증서 재발급을 요청하거나, 관리자 페이지의 **인증서 관리 → 인증서 발급** 버튼을 사용하세요. 기존 인증서는 자동으로 대체 처리됩니다.

**Q. 서버를 재시작하면 데이터가 사라져요.**  
A. 현재는 인메모리 방식으로 동작합니다. 영구 저장이 필요하다면 `utils/db.js`를 MySQL, SQLite, MongoDB 등 실제 DB로 교체해야 합니다.

**Q. 얼굴인식 정확도를 높이거나 낮추려면?**  
A. `routes/face.js` 파일의 `THRESHOLD` 값을 조정하세요. 값이 낮을수록 더 엄격하게 인증합니다.
```js
const THRESHOLD = 0.45; // 기본값, 0.35~0.55 사이에서 조정
```

**Q. 인증서 유효기간을 변경하려면?**  
A. `utils/certUtils.js`에서 `setFullYear(year + 1)` 부분을 수정하세요.
